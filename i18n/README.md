# i18n — English + Bahasa Indonesia

The site ships in **English** (`/`, `/events`) and **Bahasa Indonesia**
(`/id/`, `/id/events`). No framework, no bundler — a ~60-line vanilla
runtime plus a Node build step that pre-renders the Indonesian pages.

## How it fits together

| File | Role |
|---|---|
| `en.json` / `id.json` | **Source of truth.** Namespaced strings. `id` mirrors `en` key-for-key. |
| `strings.js` | **Generated** (`window.__I18N_STRINGS`). The runtime catalog for JS-generated text. Do not edit. |
| `i18n.js` | Runtime: `t(key)`, `Intl` formatters (`formatNumber`/`formatDate`/`formatCurrency`/`countryName`), the EN/ID toggle, `setLanguage()`. |
| `../scripts/build-i18n.mjs` | Reads `*.json`, writes `strings.js` **and** `/id/*.html` (every `[data-i18n]` swapped to Bahasa, `<html lang="id">`, canonical). |

`strings.js` and `/id/` are git-ignored and regenerated by the build
(`vercel.json` build command, and `start.command` for local dev).

## The two ways text gets translated

1. **Static HTML** → tag it and the build swaps it for the `/id/` pages:
   ```html
   <h2 data-i18n="club.title">Brotherhood,<br/>built to compete.</h2>
   <input data-i18n-attr="placeholder:join.phNationality" placeholder="e.g. …" />
   <meta data-i18n-attr="content:hmeta.desc" content="…" />
   ```
   The English text stays inline (it's what `/` serves); `id.json[key]` replaces
   it on `/id/`. Values may contain HTML (`<br/>`, `<span class='gold'>…`) and
   `data-cfg` spans — just **don't nest another `data-i18n` inside a `data-i18n`
   value** (inline the text instead).

2. **JS-generated text** → call the global `t()`:
   ```js
   el.textContent = t('squad.empty');
   el.textContent = t('hero.onSquadOther', { n: 5 });   // {n} interpolation
   ```

## Adding or changing a string — the loop

1. Add the key to **both** `en.json` and `id.json` (same path, same shape).
2. Reference it: `data-i18n="…"` / `data-i18n-attr="attr:key"` in HTML, or `t('…')` in JS.
3. Rebuild: `node scripts/build-i18n.mjs` (or just run `start.command`).
4. The build **warns** if a key is missing from `id.json` or out of parity — keep it quiet.
5. Audit: `node scripts/scan-i18n.mjs` lists any visible text *not* wrapped in
   `data-i18n` (exits non-zero if it finds untranslated copy). A new brand/proper
   noun that should stay as-is goes in that script's `ALLOW` list.

If you change the **English** copy of an already-tagged element, update
`en.json` + `id.json` too, or the `/id/` page keeps the old translation.

## Numbers, dates, currency

Use the locale-aware formatters, not hardcoded strings — that's the half that
makes it real i18n:

```js
UBJ_I18N.formatCurrency(1500000, 'IDR'); // "IDR 1,500,000" (en) / "Rp 1.500.000" (id)
UBJ_I18N.countryName('DE', 'Germany');   // "Germany" (en) / "Jerman" (id)
```
The tournament entry fee is a **number** (`CONFIG.feeAmount` in `tournament.js`)
formatted per-locale at render time.

## Language routing

- First visit: a tiny inline `<head>` script reads `navigator.language`; an
  Indonesian browser lands on `/id/` (zero-flash, runs before paint). The saved
  choice always wins over auto-detect.
- The EN/ID toggle switches **in place** — `setLanguage()` rewrites every
  `[data-i18n]`/`[data-i18n-attr]` node on the current page, updates
  `<html lang>`/`<title>`, persists the choice, and fires a document
  `i18n:changed` event (no navigation, scroll position kept). JS-rendered
  content re-renders by listening for that event — squad rows, the globe panel,
  the tournament fee, and the modals all do. On a later reload the inline
  redirect routes to the matching per-locale URL.
- `hreflang` + per-locale `canonical` tags let Google index both independently.

### Re-rendering JS content on toggle

If you add new JS that renders translated text, make it re-run on toggle:
```js
document.addEventListener('i18n:changed', () => { /* re-render with t() */ });
```
And if your code caches an element that lives **inside** a `[data-i18n]` value
(its inner HTML is replaced on swap, e.g. `#join-step-num`), re-acquire it with
`getElementById` in that listener — the old reference is detached.

## Not translated (by design)

Brand/proper nouns: "Ubud Jungle Ballers", "King of the Jungle", "Welcome to the
Jungle", "Liga Raya", sponsor names, player names, "SALAM HIJAU". Position codes
(GK/DEF/MID/FWD) and day/stat glyphs stay as-is too.
